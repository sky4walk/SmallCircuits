λ-Kalkül, erklärt von vorne
λ

Eine Maschine, die nichts kann außer ersetzen.

Lambda-Kalkül ist eine der einfachsten „Programmiersprachen" überhaupt – sie hat nur drei Bauteile. Trotzdem steckt in ihr alles, was ein Computer jemals berechnen kann: Zahlen, Wahrheitswerte, Schleifen. Diese Seite erklärt dir das in einfachen Worten, und am Ende kannst du selbst damit herumspielen.

Los geht's ↓

Teil 1Was überhaupt ist eine Funktion?

Stell dir einen Pfandautomaten vor, so einen am Supermarkteingang. Du steckst eine Flasche rein, der Automat macht immer dasselbe damit, und am Ende kommt ein Bon raus.

Eingabe🍾 Flasche
→
Maschinef
→
Ausgabe🧾 Bon

Genau das ist eine Funktion: eine feste Vorschrift, die aus einer Eingabe eine Ausgabe macht. In der Schule schreibt man das vielleicht so: f(x) = x + 1 – „nimm x, addiere 1 dazu". Im Lambda-Kalkül schreibt man dieselbe Idee einfach etwas anders:

f(x) = x + 1
≈
λx. x + 1

Mehr ändert sich erstmal gar nicht. Das λ (sprich: „Lambda") ersetzt nur das Wort f(x) =. Es ist im Grunde das griechische „Hier kommt eine Funktion!"-Symbol.

Notiz am Rand Warum ausgerechnet der griechische Buchstabe λ? Der Logiker Alonzo Church, der das System 1936 erfunden hat, brauchte einfach irgendein Symbol, das „hier wird ein Parameter gebunden" bedeutet. Es hätte auch ein Smiley sein können – λ hat sich halt durchgesetzt.

Teil 2Drei Bauteile – mehr gibt es nicht

Das Verrückte am Lambda-Kalkül: Es gibt keine Zahlen, kein +, kein if, keine Schleifen eingebaut. Nur diese drei Dinge:

Bauteil 1
x

Variable – einfach ein Name. Ein Platzhalter für irgendetwas.

Bauteil 2
λx. ⬚

Funktion (Abstraktion) – eine namenlose Mini-Maschine mit einem Parameter x.

Bauteil 3
f a

Anwendung – wende die Funktion f auf den Wert a an. Einfach hintereinanderschreiben.

Das war's. Wirklich. Schauen wir uns Bauteil 2 noch genauer an, am Beispiel λx. x + 1 (das +1 ist hier nur zur Veranschaulichung – „echtes" Lambda-Kalkül baut sich Zahlen später selbst, siehe ganz unten):

λx. x + 1
λ

Das λ sagt: „Achtung, jetzt kommt eine Funktion." Wie das Wort function in JavaScript oder def in Python.

x

Der Parameter direkt nach dem λ. Das ist der Name, der später durch einen echten Wert ersetzt wird.

.

Der Punkt trennt den Parameter vom Rest. „Ab hier beginnt der Funktionskörper."

x + 1

Der Körper (Body) – was mit dem Parameter gemacht wird, sobald ein echter Wert eingesetzt ist.

Eine Funktion mit mehreren Parametern wie f(x,y) = x schreibt man als verschachtelte Funktionen: λx. λy. x – eine Funktion, die eine Funktion zurückgibt. Das wirkt erstmal unnötig kompliziert, ist aber praktisch: So braucht man am Ende nur das eine Werkzeug „Funktion mit einem Parameter", um alles zu bauen.

Teil 3Wie „rechnet" man damit eigentlich?

Rechnen heißt im Lambda-Kalkül: einsetzen. Das nennt man β-Reduktion (Beta-Reduktion) – und es ist die einzige Rechenregel, die es überhaupt gibt.

Die Regel ist simpel: Steht eine Funktion λx. Körper direkt vor einem Wert a, dann ersetzt du im Körper jedes x durch a – und lässt das λ und den Parameternamen weg.

(λx. x) Banane
↓ β-Reduktion: ersetze x durch Banane
Banane

Die Funktion λx. x heißt übrigens Identität – sie gibt einfach das zurück, was reinkommt. Hier ein Beispiel, bei dem tatsächlich etwas passiert, eine „Echo-Maschine", die ihre Eingabe verdoppelt:

(λx. x x) Hallo
↓ β-Reduktion: jedes x im Körper wird zu Hallo
Hallo Hallo

Wenn ein Ausdruck mehrere Stellen hat, an denen man einsetzen könnte, reduziert man Schritt für Schritt – immer die äußerste, am weitesten links stehende Klammer zuerst. Man macht das so lange, bis nichts mehr einzusetzen ist. Das Ergebnis heißt dann Normalform.

Notiz am Rand Manchmal hört das Einsetzen nie auf! Der Ausdruck (λx. x x) (λx. x x) setzt sich ewig selbst wieder in sich selbst ein. Das ist kein Bug – genau diese Möglichkeit zur Endlosschleife macht das Lambda-Kalkül so mächtig wie einen ganzen Computer. Probier's unten im Spielplatz mit dem Knopf „Endlosschleife" aus (keine Sorge, die Seite stoppt nach ein paar Schritten von selbst).

Teil 5 — BonusDer eigentliche Trick: Zahlen aus dem Nichts

Oben stand: Es gibt keine eingebauten Zahlen oder Wahrheitswerte. Wie soll man damit dann irgendwas Sinnvolles berechnen? Die Antwort ist der eigentliche „Aha"-Moment des Lambda-Kalküls: Man baut sie sich aus Funktionen.

Wahr und Falsch sind einfach zwei Funktionen mit zwei Parametern, die sich entscheiden, welchen sie zurückgeben:

TRUE = λx.λy. x
FALSE = λx.λy. y

„Wahr" wählt immer die erste Option, „Falsch" immer die zweite. Mehr braucht ein if nicht zu sein! Damit lassen sich UND, ODER und NICHT bauen (probier die Bausteine UND und NICHT im Spielplatz oben aus).

Zahlen (sogenannte „Church-Zahlen") funktionieren ähnlich überraschend: Eine Zahl n ist eine Funktion, die eine andere Funktion f genau n-mal hintereinander anwendet:

0 = λf.λx. x
1 = λf.λx. f x
2 = λf.λx. f (f x)

Die „2" ist also wörtlich die Anweisung „wende f zweimal auf x an". Mit ein paar weiteren Funktionen (Nachfolger und Addition – ebenfalls im Spielplatz als Bausteine) kann man damit tatsächlich rechnen: 1 + 1 reduziert sich Schritt für Schritt wirklich zu 2. Probier's aus!

Notiz am Rand Das ist keine Spielerei: Man kann zeigen, dass sich mit diesen drei einfachen Bauteilen alles berechnen lässt, was auch ein moderner Computer berechnen kann – jede Schleife, jede Datenstruktur, jedes Programm. Das nennt man „Turing-vollständig". Lambda-Kalkül und Alan Turings Turingmaschine sind zwei völlig verschiedene Wege, die exakt gleich mächtig sind. Funktionale Programmiersprachen wie Haskell, Teile von JavaScript oder Python (lambda-Ausdrücke!) sind direkte Nachfahren von genau dem, was du gerade ausprobiert hast.

Teil 2 — Plankalkül

Teil 6Die erste Programmiersprache der Welt

Drei Jahre bevor es überhaupt programmierbare Computer gab, hat ein deutscher Ingenieur eine komplette Programmiersprache entworfen – auf Papier, im Krieg, fast völlig allein.

Konrad Zuse baute zwischen 1936 und 1941 in Berlin die ersten frei programmierbaren Rechner der Welt (die Z1 bis Z3). Aber er wollte mehr: eine Sprache, in der man allgemein Rechenpläne aufschreiben kann – nicht nur Maschinenbefehle. Zwischen 1942 und 1945 entwarf er den Plankalkül: ein System mit Variablen, Datentypen, Bedingungen, Schleifen und sogar einer Art Unterprogrammen.

Notiz am Rand „Plankalkül" heißt wörtlich: „Kalkül für Pläne" – also ein formales System, um Rechenpläne zu beschreiben. Zuse schrieb damit unter anderem ein Schachprogramm (ja, 1945!), das allerdings nie auf einem echten Computer lief. Der Plankalkül wurde erst in den 1970ern wiederentdeckt und 2000 erstmals auf einem Computer implementiert.

Warum ist das so bemerkenswert? FORTRAN, oft als „erste Programmiersprache" bezeichnet, kam erst 1957 – zwölf Jahre nach Zuses Entwurf. Zuse hatte keinen Kontakt zur angloamerikanischen Forschung (Church, Turing), er kam völlig unabhängig auf ähnliche Ideen. Lambda-Kalkül und Plankalkül sind wie zwei parallele Erfindungen desselben Grundkonzepts: der eine abstrakt-mathematisch, der andere praktisch-ingenieursmäßig.

Teil 7Die Sprache: Variablen, Zuweisungen, Bedingungen

Plankalkül hat drei Sorten von Variablen. Man erkennt sofort, wofür eine Variable gedacht ist:

Eingabe
Z0, Z1, …

Z = Eingangsgröße. Die Werte, die das Programm von außen bekommt.

Zwischenspeicher
V0, V1, …

V = Zwischenwert. Hilfsvariablen für Zwischenergebnisse.

Ergebnis
R0, R1, …

R = Resultat. Hier stehen am Ende die Ergebnisse.

Eine Zuweisung schreibt man mit dem Pfeil => – er zeigt „von links nach rechts", also von der Berechnung zum Ziel:

Z0 + Z1 ⇒ R0
V 010
S 1.01.01.0

Die obere Zeile ist die eigentliche Rechnung. Die Zeilen V (Index) und S (Typ) darunter sind das Besondere am Plankalkül: Zuse notierte alles in einer zweidimensionalen Tabelle, damit auf einen Blick klar ist, welche Variable welchen Typ hat. Im Simulator unten nutzen wir der Einfachheit halber die einzeilige Schreibweise Z0 + Z1 => R0 – der Simulator übersetzt sie automatisch in die 2D-Darstellung.

Bedingungen schreibt man mit WENN / SONST / ENDE, Schleifen mit SOLANGE / ENDE. Alle Variablen starten bei 0. So sieht zum Beispiel die Fakultätsberechnung aus:

Z0 => V0
1 => R0
SOLANGE V0 > 0
  R0 * V0 => R0
  V0 - 1 => V0
ENDE

Teil 9Zwei Welten, ein Gedanke

Lambda-Kalkül und Plankalkül entstanden fast gleichzeitig, aber völlig unabhängig voneinander – Church in Princeton, Zuse in Berlin. Der eine ist rein funktional (alles ist eine Funktion, es gibt keine Variablen, die sich ändern), der andere ist imperativ (Befehle werden Zeile für Zeile ausgeführt, Variablen werden überschrieben). Trotzdem kann man beweisen, dass beide exakt gleich mächtig sind: Alles, was man im einen ausdrücken kann, kann man auch im anderen ausdrücken.

Diese beiden Denkweisen – funktional und imperativ – ziehen sich bis heute durch die gesamte Informatik. Haskell, Lisp und Teile von JavaScript stehen in der Tradition von Church; C, Python und Java folgen eher Zuses Stil. Wer beide Ansätze kennt, versteht Computer auf einer tieferen Ebene.

Notiz am Rand Falls du neugierig geworden bist: Alan Turings Turingmaschine (1936) ist der dritte große Weg, Berechenbarkeit zu beschreiben – mit einem endlosen Band und einem Lese-/Schreibkopf. Alle drei (Lambda-Kalkül, Plankalkül, Turingmaschine) sind äquivalent. Das ist die Church-Turing-These.
