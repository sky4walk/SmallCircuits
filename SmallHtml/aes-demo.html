Eigenständig · reines JavaScript · kein Framework

AES von Grund auf

Interaktiver Port von aes.py: dieselbe Implementierung — GF(2⁸)-Arithmetik, S-Box, Key-Expansion, CBC/CTR/GCM, GHASH, SHA-256/HMAC/PBKDF2 — als eigenständige Webseite. Der Datei-Container (AESGCM03) ist binärkompatibel: mit Python verschlüsselte Dateien lassen sich hier entschlüsseln und umgekehrt.

Lehrcode. Nicht seitenkanalresistent (Table-Lookups, datenabhängige Zeiten, reines JS im Browser). Für echten Schutz eine geprüfte Bibliothek verwenden (z. B. WebCrypto / eine auditierte Library).

Text ver- und entschlüsseln

Nutzt AES-GCM mit passwortbasierter Schlüsselableitung (PBKDF2-SHA-256), Ausgabe als Base64 — dasselbe AESGCM03-Containerformat wie die Datei-Verschlüsselung, nur ohne Datei drumherum.

Datei ver- und entschlüsseln

Erzeugt bzw. liest den binären AESGCM03-Container (Magic · Schlüssellänge · Iterationen · Salt · Nonce · Tag · Chiffretext) — identisch zum Format aus aes.py encrypt/decrypt.

Beim Entschlüsseln werden Schlüssellänge und Iterationszahl automatisch aus dem Datei-Header gelesen.

Rundenansicht — ein Block Schritt für Schritt

Verschlüsselt genau einen 16-Byte-Block und zeigt den 4×4-State nach jeder Teiltransformation (SubBytes → ShiftRows → MixColumns → AddRoundKey). Geänderte Bytes gegenüber dem vorherigen Schritt sind hervorgehoben.

GCM-Trace — Zwischenwerte einer Verschlüsselung

Protokolliert H, J0, die GHASH-Zwischenwerte nach jedem 128-Bit-Block sowie den finalen Tag — entspricht der --trace-Option von aes.py.

Selbsttest — Referenzvektoren

Prüft die JS-Implementierung gegen FIPS-197-Blockvektoren (AES-128/192/256), NIST SP 800-38A CTR, den McGrew/Viega-GCM-Testfall, RFC-7914-PBKDF2, SHA-256 sowie einen vollständigen Container-Roundtrip — dieselben Vektoren, die auch aes.py selftest prüft.
