Lektion 00 · Start

Dein Setup ⚙️

Was du brauchst, wie du eigenen Code schreibst — und wie du direkt im Browser startest.

Bevor wir in die Sprache einsteigen: Lass uns klären, wie du als Entwickler arbeitest. Die gute Nachricht — JavaScript läuft direkt im Browser. Du brauchst keinen Compiler, kein Framework, keine Cloud. Nur einen Texteditor und Chrome oder Firefox.

📝
Schritt 1
Einen Code-Editor installieren

Ein Code-Editor ist wie Word — aber für Programmiercode. Er hebt Schlüsselwörter farbig hervor, zeigt Fehler direkt an und vervollständigt Code automatisch. Die beste Wahl für Einsteiger:

  • VS Code (empfohlen) — kostenlos, von Microsoft, läuft auf Windows/Mac/Linux. Download: code.visualstudio.com
  • Notepad++ — ultra-leicht, gut für Windows-Einsteiger

Nach der Installation: VS Code öffnen → File → New File → als index.html speichern → fertig.

🗂️
Schritt 2
Deine erste eigene Datei

Erstelle einen Ordner (z.B. mein-projekt) und darin eine Datei index.html. Das ist die Grundstruktur, die du immer brauchst:

index.html — kopiere das in deinen Editor!
— Datei im Editor speichern → im Browser öffnen (Doppelklick auf index.html) —
📌 Workflow
Speichere im Editor (Strg+S) → öffne die Datei im Browser (Doppelklick) → drücke F5 nach jeder Änderung. Das ist dein Basis-Workflow!
🔍
Schritt 3
Die Browser-Konsole — dein Debugger

Die Entwicklertools des Browsers sind dein wichtigstes Werkzeug. So öffnest du sie:

  • Windows/Linux: F12 oder Strg + Shift + I
  • Mac: Cmd + Option + I
  • Dann auf den Tab Console klicken

In der Konsole siehst du alle console.log()-Ausgaben deines Codes — und auch Fehlermeldungen, wenn etwas schiefläuft. Du kannst dort auch direkt JavaScript eintippen!

Live Server — Auto-Reload

In VS Code: Extensions-Symbol in der Seitenleiste → "Live Server" suchen → Install. Danach Rechtsklick auf index.html → "Open with Live Server". Der Browser lädt jetzt automatisch neu beim Speichern!

Lektion 01 · Einführung

Was ist JavaScript?

Die Sprache, die das Web zum Leben erweckt — und wie du sie zum ersten Mal verwendest.

Stell dir eine Webseite wie ein Haus vor. HTML ist das Mauerwerk — es gibt dem Haus seine Struktur: Wände, Türen, Fenster. CSS ist die Inneneinrichtung — Farben, Tapeten, Möbel. Und JavaScript ist die Elektrik und die Mechanik: die Türklingel reagiert auf Druck, das Licht geht an wenn man reinkommt, die Heizung regelt sich automatisch.

JavaScript macht Webseiten interaktiv und lebendig. Ohne JS wären Webseiten nur statische Dokumente — wie ein gedrucktes Buch. Mit JS werden sie zu Programmen: Google Maps berechnet Routen, Facebook zeigt neue Posts ohne Neuladen, YouTube spielt Videos ab. Das alles ist JavaScript.

Das Besondere: JS läuft direkt im Browser. Du musst nichts installieren, nichts kompilieren. Schreibe Code → speichere → lade im Browser → fertig. Das macht es zur perfekten ersten Programmiersprache.

💡 Was ist überhaupt ein Programm?
Ein Programm ist eine Liste von Anweisungen, die der Computer Schritt für Schritt von oben nach unten abarbeitet. Du als Programmierer schreibst diese Anweisungen — der Computer führt sie exakt so aus, wie du es schreibst. Kein mehr, kein weniger. Deshalb ist Präzision beim Programmieren so wichtig.

Deine erste Anweisung: console.log() — das gibt eine Nachricht in die Browser-Konsole aus. Der Text in den Anführungszeichen wird genau so ausgegeben, wie du ihn schreibst. Klicke auf ▶ Ausführen und schau, was passiert:

Jede Zeile mit console.log() ist eine eigene Anweisung. Der Computer führt sie der Reihe nach aus — erst Zeile 1, dann Zeile 2, dann Zeile 3. Diese Reihenfolge nennt man sequentielle Ausführung, und sie ist das Grundprinzip jedes Programms.

Probiere es selbst: Ändere den Text in einer Zeile, füge eine neue Zeile hinzu, und führe den Code erneut aus. Du kannst nichts kaputt machen!

🚨 Häufiger Anfängerfehler
JavaScript unterscheidet Groß- und Kleinschreibung. console.log() funktioniert. Console.Log() oder CONSOLE.LOG() führen zu einem Fehler. Das Gleiche gilt für alle Wörter, die du in JS schreibst — immer auf die genaue Schreibweise achten.

Lektion 02 · Grundlagen

Variablen

Variablen sind der Speicher deines Programms — wie beschriftete Schubladen für Werte.

letconstvar

Stell dir vor, du hast ein Notizbuch. Du schreibst auf einem Zettel "Name: Andre" und kannst diesen Zettel immer wieder anschauen oder überschreiben. Genau so funktioniert eine Variable in einem Programm — ein benannter Behälter, in dem du einen Wert speicherst und ihn später wiederverwenden kannst.

Ohne Variablen müsstest du jeden Wert immer wieder neu eintippen. Mit Variablen schreibst du ihn einmal, gibst ihm einen Namen, und verwendest den Namen überall im Code. Ändert sich der Wert, reicht eine einzige Änderung.

In JavaScript gibt es zwei moderne Wege, eine Variable anzulegen:

  • const — für Werte, die sich nicht ändern sollen (const = constant = konstant)
  • let — für Werte, die sich ändern können (let = "sei" oder "lass")

Das Anlegen einer Variable nennt man Deklaration. Das Zuweisen eines Wertes heißt Initialisierung. Beides zusammen sieht so aus:

Ein wichtiger Denkfehler am Anfang: Das = in JavaScript bedeutet nicht "ist gleich" wie in der Mathematik. Es bedeutet "bekommt den Wert rechts davon zugewiesen". Deshalb ist punkte = punkte + 5 gültig — lies es als: "Der neue Wert von punkte ist der alte Wert von punkte plus 5."

📌 Faustregel: wann const, wann let?
Verwende immer erst const. Wenn du merkst, dass du den Wert später ändern musst, wechsle zu let. Das alte var wirst du in älterem Code sehen — in neuem Code vermeide es, es hat unerwartetes Verhalten.

Benennungsregeln für Variablennamen: Nur Buchstaben, Zahlen, _ und $ erlaubt. Darf nicht mit einer Zahl beginnen. Keine Leerzeichen. JavaScript-Konvention ist camelCase: erstes Wort klein, jedes weitere Wort beginnt mit Großbuchstaben — z.B. meinName, anzahlDerPunkte, istAngemeldet. Wähle immer aussagekräftige Namen!

Lektion 03 · Grundlagen

Datentypen

Nicht alle Werte sind gleich — JS unterscheidet Text, Zahlen, Ja/Nein und mehr.

stringnumberbooleannullundefined

Warum brauchen wir verschiedene Datentypen? Weil der Computer unterschiedliche Werte unterschiedlich behandelt. Mit Zahlen kann man rechnen, mit Text nicht. 5 + 3 ergibt 8 — aber "5" + "3" ergibt "53" (zwei Texte werden aneinandergehängt). Der Typ eines Werts bestimmt, was du damit machen kannst.

⚠️ === vs == — warum ist das wichtig?
Das ist einer der häufigsten Bugs in JavaScript. 5 == "5" gibt true zurück, weil JS heimlich den String in eine Zahl umwandelt. 5 === "5" gibt false, weil es auch den Typ vergleicht. Verwende immer === für Vergleiche.

Lektion 05 · Logik & Ablauf

Bedingungen

Programme müssen Entscheidungen treffen — genau wie wir im Alltag.

Bisher hat dein Code immer die gleichen Schritte in der gleichen Reihenfolge ausgeführt. In der Realität möchtest du aber, dass dein Programm auf verschiedene Situationen unterschiedlich reagiert — genau wie im echten Leben: "Wenn es regnet, nehme ich einen Schirm. Sonst nicht."

Das ist bedingte Logik — der Computer trifft eine Entscheidung basierend auf einem Wert oder einer Bedingung. In JavaScript schreibst du das mit if, else if und else.

Die Grundstruktur: if (Bedingung) { Code wird ausgeführt, wenn Bedingung true ist }. Die Bedingung in den runden Klammern muss immer zu true oder false auswertbar sein.

Lektion 06 · Logik & Ablauf

Schleifen

Wiederhole Code so oft wie nötig — ohne ihn zehnmal hinzuschreiben.

forwhilefor...of

Stell dir vor, du möchtest die Zahlen 1 bis 100 ausgeben. Ohne Schleifen müsstest du 100 Zeilen console.log() schreiben. Mit einer Schleife brauchst du genau 3 Zeilen — der Computer wiederholt den Code so oft wie du sagst.

Schleifen sind eines der mächtigsten Konzepte in der Programmierung. Fast alles Interessante passiert in Schleifen: Elemente einer Liste verarbeiten, Daten transformieren, Animationen berechnen.

Die for-Schleife ist die häufigste. Sie hat drei Teile in den runden Klammern, getrennt durch Semikolons:

  • Start: let i = 0 — eine Zählvariable wird angelegt
  • Bedingung: i < 5 — solange das true ist, läuft die Schleife
  • Schritt: i++ — nach jedem Durchlauf wird i um 1 erhöht

Lektion 07 · Bausteine

Funktionen

Benannte, wiederverwendbare Code-Blöcke — das Herzstück jeder Anwendung.

Stell dir vor, du machst jeden Morgen Kaffee — immer die gleichen Schritte: Kaffeepulver abmessen, Wasser kochen, aufbrühen, warten. Statt diese Schritte jeden Morgen komplett neu zu denken, hast du eine mentale "Kaffee kochen"-Routine, die du einfach abrufst.

Genau das ist eine Funktion in JavaScript: ein benannter Block von Code, den du einmal definierst und dann beliebig oft aufrufen kannst — mit verschiedenen Eingaben (Parametern) und einer Ausgabe (Rückgabewert).

Ohne Funktionen wärst du gezwungen, den gleichen Code immer wieder zu kopieren. Das führt zu Fehlern und macht Änderungen zum Albtraum. Mit Funktionen schreibst du Code einmal und verwendest ihn überall.

Arrays

Geordnete Listen — und die mächtigen Methoden, sie zu verarbeiten.

Was, wenn du nicht eine Stadt, sondern zehn Städte speichern willst? Zehn einzelne Variablen anzulegen wäre umständlich — und was, wenn es hundert werden? Dafür gibt es Arrays: eine Variable, die mehrere Werte in einer geordneten Liste speichert.

Lektion 10 · Praxis

DOM & HTML steuern

JavaScript greift ins HTML ein — und verändert die Seite in Echtzeit.

Bisher hat dein JavaScript nur Werte berechnet und in die Konsole geschrieben. Der eigentliche Einsatzzweck ist ein anderer: HTML-Seiten verändern. Wenn du auf einer Webseite auf einen Button klickst und etwas passiert — ein Menü klappt auf, Text erscheint, eine Farbe ändert sich — dann macht das JavaScript über den DOM.

DOM steht für Document Object Model. Wenn der Browser deine HTML-Datei lädt, wandelt er jedes HTML-Tag in ein JavaScript-Objekt um. Das gesamte Dokument wird zu einer Baumstruktur von Objekten, die du mit JS lesen und verändern kannst. Das ist der DOM.

Lektion 11 · Praxis

Events

Reagiere auf Nutzeraktionen — der Code läuft erst, wenn etwas passiert.

Bisher hat dein Code immer sofort beim Laden der Seite gestartet und von oben nach unten alles ausgeführt. Das ist für viele Dinge nützlich — aber interaktive Webseiten müssen auf den Nutzer warten und dann reagieren.

Ein Event ist ein Ereignis, das passiert: ein Klick, eine Tastatureingabe, eine Mausbewegung, das Laden der Seite, ein Formular das abgesendet wird. Mit addEventListener() sagst du dem Browser: "Wenn dieses Event passiert, führe diesen Code aus."

Das Besondere: Der Code in einem Event-Listener wird nicht sofort ausgeführt — er wartet, bis das Event eintritt. Du hinterlässt quasi eine Notiz für den Browser: "Wenn der Button geklickt wird, ruf diese Funktion auf."

    Lektion 13 · Weiterführend

    Fehlerbehandlung

    Fehler passieren immer — lerne, wie du sie abfängst, verstehst und elegant behandelst.

    Kein Programm ist fehlerfrei. Server sind nicht erreichbar, Nutzer machen unerwartete Eingaben, Daten haben das falsche Format. Ohne Fehlerbehandlung bricht dein Programm einfach ab — mit Fehlerbehandlung kannst du sinnvoll reagieren, dem Nutzer eine verständliche Meldung geben, und das Programm am Laufen halten.

    JavaScript stoppt bei einem Fehler sofort — alle folgenden Zeilen werden nicht ausgeführt. Mit try/catch fängst du diesen Absturz ab. Alles im try-Block wird ausgeführt. Passiert ein Fehler, springt JavaScript sofort in den catch-Block — der Rest von try wird übersprungen. catch empfängt ein Error-Objekt mit zwei wichtigen Eigenschaften: .message (was schiefging) und .name (welcher Fehlertyp).

    Der optionale finally-Block läuft immer — egal ob Fehler oder nicht. Er ist ideal für Aufräumarbeiten (Datenbankverbindung schließen, Ladeindikator ausblenden etc.).

    Lektion 14 · Weiterführend

    Scope & Closures

    Wo sind Variablen sichtbar — und was ist eine Closure?

    Scope (Sichtbarkeitsbereich) bestimmt, von wo aus eine Variable zugreifbar ist. Stell dir Scope wie verschachtelte Räume vor: Wer in einem inneren Raum ist, sieht alles im äußeren Raum — aber nicht umgekehrt.

    In JavaScript gibt es drei Scope-Ebenen. Der globale Scope ist überall sichtbar. Der Funktions-Scope gilt innerhalb einer Funktion — Variablen dort sind von außen unsichtbar. Der Block-Scope gilt innerhalb von {} — let und const respektieren das, das alte var nicht.

    Asynchrones JavaScript & Promises

    JavaScript wartet nicht — oder doch? Wie dein Code mit Zeit umgeht.

    JavaScript ist single-threaded — es kann immer nur eine Sache gleichzeitig tun. Stell dir einen Kellner vor, der nur einen Tisch bedient: er nimmt die Bestellung auf, geht in die Küche, wartet dort auf das Essen, kommt zurück, serviert es — und erst danach ist der nächste Tisch dran. Das wäre sehr ineffizient.

    In der Realität gibt der Kellner die Bestellung in der Küche ab und bedient inzwischen andere Tische — wenn das Essen fertig ist, bringt er es. Das ist asynchrones Programmieren: statt zu warten, macht der Code in der Zwischenzeit etwas anderes.

    Ein Promise (Versprechen) repräsentiert einen Wert, der noch nicht da ist, aber in der Zukunft kommen wird. Ein Promise hat drei Zustände: pending (wartend), fulfilled (erfüllt), rejected (fehlgeschlagen).

    Lektion 19 · Fortgeschritten

    async / await

    Asynchronen Code so schreiben, als wäre er synchron — die moderne Art.

    .then().catch()-Ketten können bei vielen aufeinanderfolgenden asynchronen Operationen unleserlich werden. async/await ist syntaktischer Zucker, der den gleichen Code viel lesbarer macht — er sieht aus wie normaler, sequentieller Code, ist aber immer noch asynchron.

    async vor einer Funktion macht sie zu einer async-Funktion — sie gibt immer automatisch ein Promise zurück. Innerhalb einer async-Funktion kannst du await vor jedes Promise schreiben — der Code pausiert an dieser Stelle, bis das Promise erfüllt wird, und gibt dann den Wert zurück. Der Rest des Programms wird währenddessen nicht blockiert.

    Fehlerbehandlung bei async/await: statt .catch() verwendest du das bekannte try/catch. Das ist besonders praktisch, weil du synchrone und asynchrone Fehler gleichzeitig abfangen kannst.

    fetch() & APIs

    Echte Daten aus dem Internet laden — dein Code spricht mit der Welt.

    Bisher hat dein Code nur mit Daten gearbeitet, die du selbst hingeschrieben hast. In der Praxis kommen Daten aus dem Internet: Wetterdaten, Nutzerprofile, Produktlisten, Nachrichten. Diese Daten liefern APIs (Application Programming Interface) — Schnittstellen, über die Server Daten bereitstellen.

    Eine API ist wie ein Menü im Restaurant: du bestellst (sendest einen HTTP-Request an eine URL), und der Server liefert die Antwort (meist als JSON). JSON steht für JavaScript Object Notation — es ist ein Textformat, das genauso aussieht wie JavaScript-Objekte, und von Menschen wie Maschinen leicht lesbar ist.

    fetch(url) sendet einen HTTP-GET-Request und gibt ein Promise zurück. Die Antwort enthält das Response-Objekt — daraus holst du die Daten mit .json(), was ebenfalls ein Promise zurückgibt. Daher zwei await-Aufrufe.

    Lektion 21 · Praxis II

    localStorage

    Daten dauerhaft im Browser speichern — auch nach einem Neuladen noch da.

    Variablen in JavaScript existieren nur so lange, wie die Seite offen ist. Lädst du die Seite neu, sind alle Werte weg. localStorage ist ein einfacher Schlüssel-Wert-Speicher im Browser, der diese Einschränkung aufhebt: gespeicherte Daten bleiben erhalten, bis der Nutzer sie löscht — auch nach Neuladen, Tab-Schließen und Browser-Neustart.

    Wichtig: localStorage speichert nur Strings. Um Objekte oder Arrays zu speichern, musst du sie vorher mit JSON.stringify() in einen String umwandeln, und beim Lesen mit JSON.parse() zurückwandeln.

    localStorage ist ideal für: Benutzereinstellungen (Sprache, Theme), den letzten Stand einer Eingabe, einfache App-Daten wie eine To-Do-Liste. Für sensible Daten (Passwörter, Tokens) ist localStorage nicht geeignet — dort lieber Cookies mit httpOnly-Flag.

    Formularvalidierung

    Nutzereingaben prüfen, bevor sie abgesendet werden — sicher und benutzerfreundlich.

    Formulare sind ein Kernstück jeder Web-App: Anmeldung, Kontaktformulare, Bestellungen. Der Browser hat eingebaute Validierung (z.B. required, type="email"), aber für echte Kontrolle und benutzerfreundliche Fehlermeldungen braucht es JavaScript.

    Das wichtigste Event bei Formularen ist submit — es feuert, wenn der Nutzer absendet. event.preventDefault() verhindert das Standard-Verhalten (Seite neu laden). Dann kannst du die Eingaben selbst prüfen und entscheiden, was passiert.

    Gute Validierung prüft: Ist das Pflichtfeld ausgefüllt? Hat die E-Mail das richtige Format? Ist das Passwort lang genug? Sie zeigt klare, hilfreiche Fehlermeldungen — und lässt das Formular nur absenden, wenn alles korrekt ist.

    Debugging

    Fehler finden und verstehen — die wichtigste Fähigkeit eines Entwicklers.

    Als Anfänger glaubt man, Programmieren heißt Code schreiben. In Wirklichkeit verbringen erfahrene Entwickler genauso viel Zeit damit, Fehler zu suchen (Debugging) wie neuen Code zu schreiben. Das ist normal — und je besser du im Debugging wirst, desto schneller kommst du voran.

    JavaScript-Fehler passieren in drei Kategorien: Syntaxfehler (Tippfehler im Code — Browser lädt die Datei gar nicht), Laufzeitfehler (Code-Syntax korrekt, aber beim Ausführen schlägt etwas fehl — z.B. null.property) und Logikfehler (Code läuft ohne Fehler, produziert aber falsche Ergebnisse — die schwierigsten!)

    Dein wichtigstes Werkzeug ist die Browser-Konsole (F12 → Console). Dort siehst du alle Fehler mit Dateiname und Zeilennummer. Das console-Objekt bietet mehr als nur .log():

    Node.js & npm

    JavaScript außerhalb des Browsers — auf dem Server, in der Kommandozeile, überall.

    Bisher läuft dein JavaScript immer im Browser. Node.js macht JavaScript auch außerhalb des Browsers ausführbar — direkt auf deinem Computer, auf Servern, in Build-Tools. Node.js baut auf der gleichen V8-Engine auf wie Chrome, hat aber statt DOM-APIs Systemzugriff: Dateien lesen/schreiben, Netzwerkserver starten, Datenbanken ansprechen.

    npm (Node Package Manager) ist der Paketmanager für JavaScript. Über npm kannst du aus einem riesigen Repository von über 2 Millionen Paketen installieren — Bibliotheken, die andere Entwickler geschrieben haben, damit du nicht alles selbst bauen musst. package.json ist die Konfigurationsdatei deines Projekts: sie listet alle Abhängigkeiten auf, damit andere das Projekt mit einem Befehl zum Laufen bringen können.

    Modernes JavaScript verwendet ES-Module mit import/export statt des älteren CommonJS mit require(). In Node.js und in Browser-Projekten mit Build-Tool ist import/export heute Standard.
